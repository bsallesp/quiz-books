# Governança de Documentação (32-34)

32. **SDD (System Design Document):** Mantenha o arquivo `SDD.md` atualizado com a arquitetura, tecnologias e fluxos principais do sistema. Qualquer mudança arquitetural deve ser refletida aqui.
33. **Backlog:** Use `backlog.md` para rastrear todas as tarefas pendentes, bugs e melhorias. Nada deve ser implementado sem estar listado aqui.
34. **Histórico:** Ao concluir tarefas, mova-as do backlog para `history.md`, registrando a data e um breve resumo do que foi feito.
