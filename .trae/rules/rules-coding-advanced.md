# Boas Práticas Avançadas (8-15)

8. Valide e sanitize todas as entradas externas.
9. Fuja de “números mágicos”; use constantes nomeadas.
10. Trate exceções de forma específica, nunca as silencie sem motivo.
11. Mantenha o código desacoplado com injeção de dependência e interfaces.
12. Observe o princípio da menor surpresa: comportamento deve ser intuitivo.
13. Prefira composição ao invés de herança quando possível.
14. Faça code review obrigatoriamente antes de mergear para main.
15. Documente APIs públicas com exemplos de uso.
