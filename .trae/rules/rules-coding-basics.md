# Boas Práticas Básicas (1-7)

1. Escolha nomes de variáveis, funções e classes claros e autoexplicativos.
2. Mantenha funções pequenas e com uma única responsabilidade.
3. Siga o princípio DRY (Don’t Repeat Yourself).
4. Comente o “porquê”, não o “o quê”; o código deve ser autoexplicativo.
5. Use controle de versão (Git) com mensagens de commit descritivas.
6. Escreva testes unitários antes ou junto com o código (TDD ou pelo menos cobertura razoável).
7. Automatize build, teste e deploy com integração contínua (CI/CD).
